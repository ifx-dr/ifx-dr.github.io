an Interface Design Description (IDD) can refer to a Communication Profile (CP). No other documentation document can refer to a CP. Hence the domain of this object property is the class of IDDs. An IDD can refer to a CP, but this is not mandatory. CPs are intended to be used in cases in which several services use the same CP. If the CP is unique for a specific service, the CP information can directly be included in the IDD. If the IDD refers to a CP, this must be a unique reference. That is, one IDD must not refer to different CPs. Hence this object property is functional. (If the service could be implemented using different communication technologies, each implementation would need an own IDD!!).
